Install & Build Ubuntu + Caddy

Node.js 18+ required. The MCP server lives inside the Arrissa repo.

1Install Node.js 20 LTS
bash 
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v  # should print v20.x.x
2Install dependencies & build

The mcp-server/ folder is part of the main Arrissa repo — no separate clone needed.

bash 
cd /var/www/arrissa/mcp-server
npm install
npm run build
On success npm run build exits with no errors and creates the dist/ folder. The server reads the API key and base URL directly from the SQLite database — no .env file needed.
3Run as a systemd service

Create a systemd unit so the MCP server starts on boot and restarts on crash:

bash 
sudo tee /etc/systemd/system/arrissa-mcp.service <<'EOF'
[Unit]
Description=Arrissa Data MCP Server
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/arrissa/mcp-server
ExecStart=/usr/bin/node dist/index.js
Restart=always
RestartSec=5
Environment=PORT=3000
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable arrissa-mcp
sudo systemctl start arrissa-mcp
sudo systemctl status arrissa-mcp
Check logs any time with sudo journalctl -u arrissa-mcp -f. To restart after a rebuild: sudo systemctl restart arrissa-mcp
4Expose via Caddy (recommended)

Add a route to your existing Caddyfile so the MCP endpoint is available at https://yourdomain.com/mcp-server — no extra port needed:

bash 
sudo tee -a /etc/caddy/Caddyfile <<'EOF'

# Arrissa MCP Server — add this block inside your existing yourdomain.com { } block
# or as a separate subdomain block:
mcp.yourdomain.com {
    reverse_proxy localhost:3000
    encode gzip
}
EOF

sudo systemctl reload caddy
Prefer a path route? Inside your existing yourdomain.com { } block add: handle /mcp* { reverse_proxy localhost:3000 } — then your endpoint becomes https://yourdomain.com/mcp.

After reloading Caddy, verify the server is up:

bash 
curl https://mcp.yourdomain.com/health

Install & Build Windows + XAMPP

Node.js 18+ required. Run in PowerShell as Administrator.

1Install Node.js 20 LTS

Download the Windows installer from nodejs.org and install. Then verify:

powershell 
node -v   # v20.x.x
npm -v
2Install dependencies & build
powershell 
cd C:\xampp\htdocs\arrissa\mcp-server
npm install
npm run build
3Run as a Windows Scheduled Task (auto-start)

Register a task that starts the MCP server at boot (run PowerShell as Administrator):

powershell 
$nodeExe = (Get-Command node).Source
$script  = "C:\xampp\htdocs\arrissa\mcp-server\dist\index.js"
$workDir = "C:\xampp\htdocs\arrissa\mcp-server"
$action  = New-ScheduledTaskAction -Execute $nodeExe -Argument $script -WorkingDirectory $workDir
$trigger = New-ScheduledTaskTrigger -AtStartup
$settings = New-ScheduledTaskSettingsSet -ExecutionTimeLimit 0 -RestartOnIdle $false
$env     = New-ScheduledTaskSettingsSet -ExecutionTimeLimit 0
Register-ScheduledTask -TaskName "ArrissaMCPServer" -Action $action -Trigger $trigger -Settings $settings -RunLevel Highest -Force

# Start it now without rebooting
Start-ScheduledTask -TaskName "ArrissaMCPServer"
To stop: Stop-ScheduledTask -TaskName "ArrissaMCPServer"  |  To remove: Unregister-ScheduledTask -TaskName "ArrissaMCPServer" -Confirm:$false
4Verify the server is running
powershell 
Invoke-RestMethod http://localhost:3000/health | ConvertTo-Json

Authentication

Every request to /mcp must include your API key as a header

!Required header

The MCP server reads your API key from the Arrissa database. Every request to the /mcp endpoint must carry:

http 
x-api-key: YOUR_API_KEY

Find your API key at Settings → API Key inside the Arrissa dashboard. Requests with a missing or wrong key receive 401 Unauthorized.

Keep your API key secret. Never commit it to version control or expose it client-side. Rotate it from the Settings page if it is ever leaked.

Connect Claude Desktop

Add the Arrissa MCP server to your Claude Desktop config

1Open your Claude Desktop config file

macOS / Linux: ~/.config/claude/claude_desktop_config.json

Windows: %APPDATA%\Claude\claude_desktop_config.json

Create the file if it does not exist. Then add (or merge) the mcpServers block:

2Remote server (Ubuntu + Caddy)
json 
{
  "mcpServers": {
    "arrissa-data": {
      "type": "http",
      "url": "https://mcp.yourdomain.com/mcp",
      "headers": {
        "x-api-key": "YOUR_API_KEY"
      }
    }
  }
}
If you used a path route instead of a subdomain, the URL would be https://yourdomain.com/mcp
3Local server (Windows localhost)
json 
{
  "mcpServers": {
    "arrissa-data": {
      "type": "http",
      "url": "http://localhost:3000/mcp",
      "headers": {
        "x-api-key": "YOUR_API_KEY"
      }
    }
  }
}
4Restart Claude Desktop

Quit and relaunch Claude Desktop. Open a new conversation — you should see arrissa-data in the tools panel (hammer icon). Ask Claude to "list available Arrissa tools" to confirm all 63 are loaded.

Try asking: "Get me the latest USD economic events from the Arrissa calendar" — Claude will invoke get_latest_economic_events automatically.

Connect Cursor

Use Arrissa tools directly in the Cursor AI editor

1Open Cursor MCP settings

In Cursor go to Settings → MCP (or edit ~/.cursor/mcp.json directly):

2Add the Arrissa server
json 
{
  "mcpServers": {
    "arrissa-data": {
      "url": "https://mcp.yourdomain.com/mcp",
      "headers": {
        "x-api-key": "YOUR_API_KEY"
      }
    }
  }
}

For local use replace the URL with http://localhost:3000/mcp.

3Reload & verify

Reload the Cursor window. In the chat panel (Ctrl+L) you should now see the Arrissa tools available. Test with: "What economic events are scheduled for today?"

Test the Server

Verify auth and tool discovery work correctly

1Health check (no auth required)
bash 
curl https://mcp.yourdomain.com/health

Expected response includes "status": "ok" and "tools": 63.

2MCP initialize — with auth header
bash 
curl -s -X POST https://mcp.yourdomain.com/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -H "x-api-key: YOUR_API_KEY" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "initialize",
    "params": {
      "protocolVersion": "2025-03-26",
      "capabilities": {},
      "clientInfo": { "name": "test", "version": "1.0" }
    }
  }' | jq .
3Confirm 401 without the header
bash 
curl -s -X POST https://mcp.yourdomain.com/mcp \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2025-03-26","capabilities":{},"clientInfo":{"name":"test","version":"1.0"}}}'
# Should return: {"error":"Unauthorized","message":"Provide your Arrissa API key in the x-api-key request header"}
